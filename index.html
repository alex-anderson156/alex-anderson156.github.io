Delphi Control Panel

DELPHI Control Panel is my solution to allow our current team to create, maintain and develop Applications around Delphi. The aim of this Project was to create a flexible, easy to distribute system with a common and easy to use interface for all the Applications we need to produce.

Feature List:

  • Customisable Front pages - to allow Engineers to see information they need quickly.
  • Android Inspired application Architechure - to allow developers to easily create new Applications using an Activity workflow system.
  • Tile Based Interface System - To Support both Desktop and Touch screen devices and provide a simplistic Interface for anyone to easily use.
  • Integrated Graphing API - Interfacing with OxyPlot, the graph system is designed to allow developers to effortless create Charts and graphs efficiently and quicking using the Builder Design Pattern.
  • Integrated Hierarchical Tree Rendering - To Allow visual representation of Tree Structures.
  • Email and Notifications - Notify Key Personel via E-Mail or TOAST Notifications when machines are detected to be running out of spec or into problems.

SQUAWK Box

While at DELPHI I have taken a Consulting Role on this Project. The Aim was to create an easy way for Line Operators to input Lost-Time data into our System without actually loosing time by Inputting data.

Our solution was to use a Raspberry PI and a touch screen to display a website with a simple, easy to use interface hosted using ASP.Net.

This allowed our Solution to be compact, robust, cost-effective and sit next to the line operator, allow them to continue operating the machine and having minimal to no downtime due to data input.

SQUAWK is Now being used widly accross the site with plans to implement the process on more machines. So far, there has been lots of positive feedback from both users and Managment and reports of increased productivity have been welcome.

Technology Used:
  • ASP.NET
  • WCF (Server Side)
  • Web-Languages (HTML5, Bootstrap, AngularJS)

Production Line Monitor

One of my First projects at DELPHI was to create a Production Monitor to track the output of our newest Build Line. Working from a limited requirments specification and liaising with key engineers, everyone has been impressed with the outcome of the software.

Software Features:
  • Live Status Monitoring - Notifications on machine faults.
  • Production vs. Target Analysis - Over given, weeks months and years.
  • Automated Report generation - Calculates Lost time and Production data from SQL Database and provides the engineers with information in a familiar report, allowing them to continue working on upgrading the process and fixing problems.
  • Symptom and Solution reporting and troubleshooting - Allows the engineers to specify actions to common problems, allowing for better communication and problem soling amongst the workforce.

Technology Used

  • WPF - .NET 4.5
  • SQL Server - Database design and Query Design

Live Monitoring Website

My Second large Project at Delphi was to create a system which was capable of monitoring the status of various critical process on Machinary and alert key personal when something was wrong.

This was achived by creating a website in ASP.net so that the statuses could be viewed from any machine or PC around the Site without having to download Software.

To Alert of Software or events, I Listen directly to the Event Log and passed event up to the server from a small, lightweight client installed on the machine.
